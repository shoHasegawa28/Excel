各端末のショートカットキ-をまとめるエクセルを作る

開発環境
Ecsel

目的
開発で今後より効率的にするための
ショートカットキーをまとめたツールを作成する

取り扱うデータ

ショートカットキー (以後 skとする)
skの内容
skの種類
環境

機能
1.登録
どの開発環境のskなのか
skの内容や種類は何なのかを登録する

形式、検索機能を使わずでも見やすいようにする

コンセプト　検索機能をメインとせず
見やすいフォーマットにする

2.検索
プラットフォームと項目と近似一致で特定する

例
削除　対象2件
			1.○○〇　内容1
			2,○○    内容2

データ登録フォーマット

3列　種類,キー,動作

設計

FORMから値を取得するためのメソッド
FORMから実際に書き込むための処理

表に行を追加する
行の追加
場合によって新規シート作成
更新処理

作成順序

1.入力画面作成

必要となるデータ

登録済みか新規の項目
キーの組み合わせ
実行内容

2.エクセルのシートに値を反映させる
B行目、項目
C行目、キー
D行目、内容

3.表の体裁を崩さず挿入
行の検索
行末に追加

4.同じキーの場合
更新をするかどうかを判定する
更新する場合はキーの内容変更
更新しない場合は、処理をせず終了

必要となる関数
inialaze

項目、キーのコンボボックス初期化

insertBtn_onClickEvent

入力値の取得　変数に格納
新規入力に値があれば新規項目を反映させる

項目を検索する
検索範囲B6行から行末まで

対応するキーが存在するか検索

すでに存在する場合は、更新するかどうかの確認画面表示
存在しない場合は、項目の最後に新しい行を作成し挿入する
その際、下に別の項目がある場合は、したの項目も一段下にずらす

表の体裁を更新する

必要となる変数

String RegisteredCategories　登録済み項目
String NewCategories 		  新規項目
String LeaderKey			主軸のキー
String SubKey				サブのキー
String Content				内容
Range CategorysCell		項目を入力するセル
Range Key				キーを入力するセル
Range ContentCell		

修正フェーズ
nullチェック
項目の探索
探索結果に応じて行の追加
現在の行から登録キーの探索
hitした場合は強制終了

hitしなかった場合は,次の項目の行に1行追加してキーの追加

表の体裁を整える

表の中から特定のキーを探すメソッド
hitしたらhitした行の番号を返す関数を作成
